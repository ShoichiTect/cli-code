#Roadmap (Draft)

このプロジェクトは「競合しないニッチ」へ寄せる方針。
方向性は「観測・計測」を主軸にしつつ、コード探索とGitHub連携を伸ばす。

## 北極星 (何を価値にするか)

- モデルや運用の「見えないコスト」を可視化する
- 変更や改善の効果を数値で示す
- 日常の開発フロー (探索・Issue・レビュー) を補助する

## 近い将来 (短期)

- Token消費レポート (セッション/ツール/プロバイダ別の集計)
- コスト見積り (USD/JPY, モデル単価の設定管理)
- レイテンシ計測 (応答時間/リトライ回数の記録)
- コード探索のUX改善 (検索履歴、検索フィルタのプリセット)

## 中期

- モデル機能測定スイート
  - 同一タスクを複数回実行して分散や安定性を計測
  - 失敗率/再試行率/ツール発火率を記録
- セッションレポート自動生成 (Markdownで要点/コスト/改善案)
- 無駄なツール呼び出し検出 (失敗ツール/冗長ツールの割合)

## 中期 (GitHub連携)

- ghcli連携でIssue発行 (エラー/気づき/レポートの自動起票)
- GitHub Actionsでレビュワーとして動かす
  - PR本文・diff要約
  - リスク/テストギャップ指摘
  - コスト/レイテンシの定点観測コメント

## 中期 (GUI)

- React WebAppを別プロジェクトで作成
  - CLIはagentic loopのコアとして再利用
  - 連携方式はAPI/IPC前提で設計

## 長期

- VPS/VMでの常駐運用モード
  - API化 (REST)
  - キュー/ジョブ管理
  - 永続ログ (SQLite/Postgres)
- 複数プロバイダの自動フォールバック
  - SLAやコストに応じた切り替え

## 依存・前提

- モデル単価テーブルの管理方法 (設定ファイル or 内蔵)
- トークン計測の取得タイミング (APIレスポンス依存)
- GitHub連携の認証方式 (gh auth / PAT)

## 未決事項

- どの指標を「最重要KPI」にするか
- レポートの出力先 (CLI表示のみ / ファイル保存 / Issue投稿)
